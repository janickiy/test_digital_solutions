# Million Item Selector

Двухпанельный интерфейс для выбора и сортировки элементов из набора с базовыми
ID `1…1 000 000`. Состояние выбранных элементов и их порядок хранится в памяти
Express-процесса и общее для всех посетителей.

## Запуск

Требуется Node.js 20 или новее.

```bash
npm install
npm start
```

Открыть <http://localhost:4173>. Для разработки с автоматическим перезапуском:

```bash
npm run dev
```

Проверки:

```bash
npm test
npm run check
```

## Как выполнены требования

- Базовый миллион не материализуется в памяти: ID вычисляются по курсору. Это
  оставляет расход памяти пропорциональным только числу добавленных и выбранных
  значений.
- Оба списка используют cursor pagination с жестким серверным пределом 20
  элементов на ответ. Фильтрация также сканируется по курсору и возвращает не
  более 20 совпадений.
- Пользовательские ID — непустые строки до 128 символов. Они могут быть
  числовыми, текстовыми или смешанными. Уже существующие значения не добавляются.
- DnD отправляет порядок загруженной проекции. При активном фильтре сервер
  переставляет совпавшие ID в их прежних слотах, поэтому скрытые элементы не
  меняют взаимный порядок. Для клавиатуры тот же механизм доступен стрелками на
  сфокусированном drag-маркере.
- Клиент имеет три очереди: чтения объединяются в `/api/read/batch` раз в 1 с,
  выбор/удаление/сортировка — в `/api/state/batch` раз в 1 с, добавления — в
  `/api/items/batch` раз в 10 с.
- Одинаковые чтения объединяются по ключу запроса, выбор для одного ID
  схлопывается до последнего намерения, добавления дедуплицируются по ID.
  Дополнительно сервер хранит обработанные `opId`/`requestId`, а наборы на базе
  `Set` гарантируют уникальность даже при повторной доставке HTTP-запроса.
- При обновлении вкладки ожидающие изменения отправляются через `sendBeacon`.
  Поисковые строки нигде не сохраняются.

## API

- `POST /api/read/batch` — пакетное чтение ресурсов `available`, `selected`,
  `stats`.
- `GET /api/items/available?query=&cursor=&limit=20` — доступные элементы.
- `GET /api/items/selected?query=&cursor=&limit=20` — выбранные элементы.
- `POST /api/items/batch` — пакет произвольных новых ID.
- `POST /api/state/batch` — пакет операций `select`, `deselect`, `reorder`.
- `GET /api/health` — состояние процесса и счетчики.

Все данные намеренно сбрасываются при перезапуске сервера, так как база данных
по условию не используется.

## Production

Готовые конфигурации systemd и Apache находятся в `deploy/`. Production-процесс
слушает только `127.0.0.1:4173`; внешний HTTPS завершается на Apache.
